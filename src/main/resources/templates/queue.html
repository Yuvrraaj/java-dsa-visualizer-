A Queue is a linear data structure that models real-world lines. It enforces a strict First In, First Out (FIFO) policy. The element that has been in the queue the longest is the first one to be removed.

Elements are added at the Rear (Tail) and removed from the Front (Head). Think of a printer queue, web server request handling, or customers waiting at a checkout.

FIFO Architecture

Guarantees order of processing. Essential for fair scheduling algorithms.

Memory Implementations

Can be implemented via Arrays (requires shifting or circular logic) or Linked Lists (O(1) dynamic scaling).

Time Complexity

Enqueue (Add): O(1)
Dequeue (Remove): O(1)
Peek: O(1)

Types of Queues

1. Standard / Linear Queue

The basic FIFO structure. A major drawback when implemented with simple arrays is that once the rear reaches the end of the array, you cannot add new elements even if there is empty space at the front (caused by previous dequeues).

2. Circular Queue

Solves the memory waste of linear array queues. The last position is connected back to the first position to make a circle. It uses the modulo operator (rear + 1) % size to wrap around.

3. Priority Queue

Elements are dequeued based on a priority weight rather than strict FIFO order. Usually implemented internally using a Heap data structure.
