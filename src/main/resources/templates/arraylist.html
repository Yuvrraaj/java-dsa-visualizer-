ArrayList (Dynamic Array) — Conceptual Understanding

An ArrayList is a dynamic resizable array provided by the Java Collections Framework (similar to std::vector in C++ or standard lists in Python). Unlike traditional arrays that have a strict, unchangeable capacity, an ArrayList automatically grows or shrinks as elements are added or removed.

This provides the perfect balance: you get the blazing-fast index access of an array, combined with the flexible memory sizing of a linked list.

How It Works Under the Hood

An ArrayList doesn't possess magical memory powers. Internally, it is just a standard, fixed-size array.

When you create an ArrayList, Java creates a normal array behind the scenes (usually with an initial capacity of 10). When you keep adding elements and that internal array becomes full, the ArrayList performs a Resizing Operation:

  1. It creates a brand new, larger array in memory (typically 1.5x or 2x the size of the old one).
  2. It copies all existing elements from the old array into the new array.
  3. It updates its internal reference to point to the new array, and the Garbage Collector deletes the old one.

Why ArrayLists Are Needed (Pros vs Cons)

Advantages:

  • Dynamic Growth: No need to guess the size of your data beforehand.
  • Random Access: Fetching elements via index is instant, just like standard arrays.
  • Rich API: Comes with built-in methods for sorting, reversing, checking if it contains an item, and removing items, saving you from writing boilerplate loops.

Disadvantages:

  • Resizing Overhead: The moment the array gets full, the next add() operation will be very slow because it has to copy every single element to a new memory block.
  • Primitive Types: In Java, ArrayLists can only store Objects (like Integer, String), not primitive types (like int, double). This causes memory overhead due to "Auto-boxing".

Time Complexity Analysis

  • Access element (get(i)): O(1)
  • Add at end (add(val)): O(1) Amortized (Usually instant, but occasionally O(n) when resizing triggers).
  • Insert at middle (add(i, val)): O(n) (Requires shifting elements right).
  • Delete at middle (remove(i)): O(n) (Requires shifting elements left).
