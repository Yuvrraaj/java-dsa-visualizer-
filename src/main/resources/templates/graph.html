Graphs — Network Data Structures

A Graph is a non-linear data structure made up of Vertices (Nodes) and Edges (Connections). Unlike Trees, which follow a strict hierarchy, graphs allow any node to connect with any other node, forming complex networks.

Formally, a graph is represented as: G = (V, E), where V represents vertices and E represents edges.

Graphs model real-world systems such as social networks, road maps, airline routes, and computer networks.

Types of Graphs

  • Undirected Graph: Connections work both ways. If A connects to B, you can travel from A → B and B → A.
  • Directed Graph (Digraph): Edges have direction. Movement is allowed only along the arrow direction.
  • Weighted Graph: Each edge stores a cost or distance. Used in GPS navigation and routing algorithms.
  • Unweighted Graph: All edges are treated equally (weight = 1).
  • Cyclic vs Acyclic: A cyclic graph contains loops. A Directed Acyclic Graph (DAG) has no cycles and is widely used in task scheduling and dependency resolution.

Memory Representation of Graphs

Graphs do not follow linear memory layouts, so special representations are used to store connections efficiently.

1. Adjacency Matrix

A 2D array of size V × V. If an edge exists from vertex i to vertex j, then matrix[i][j] = 1 (or stores the edge weight). Otherwise it remains 0.

  • Edge lookup: O(1)
  • Space usage: O(V²)
  • Best for: Dense graphs

2. Adjacency List

Each vertex stores a list of its neighbors. Usually implemented using an ArrayList or HashMap.

List<List<Integer>> graph = new ArrayList<>();
  • Space usage: O(V + E)
  • Efficient for: Sparse graphs
  • Neighbor traversal: Very fast
Adjacency Lists are the most commonly used representation in coding interviews and competitive programming.

Why Graphs Are Important

  • Social network recommendations
  • Navigation and shortest path systems
  • Internet routing protocols
  • Dependency resolution (build systems)
  • AI search and game pathfinding
