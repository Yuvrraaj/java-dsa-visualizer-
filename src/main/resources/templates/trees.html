Trees — Hierarchical Data Structures

While Arrays, Stacks, and Linked Lists are linear data structures (data is arranged sequentially), a Tree is a hierarchical data structure. It represents relationships where one item branches out into multiple items, much like a family tree or a computer's file system.

A Tree consists of vertices (called Nodes) and edges that connect them. It must have no cycles (no closed loops). The topmost node is called the Root.

[Image of Tree Data Structure Terminology showing root, branches, and leaves]

Core Terminology

  • Root: The absolute top node of the tree. There is only one root.
  • Edge: The connecting link between two nodes.
  • Parent & Child: A node that branches down is a Parent. The nodes it connects to are its Children.
  • Leaf: A node with NO children (the absolute bottom of a branch).
  • Sibling: Nodes that share the exact same Parent.
  • Depth: The number of edges from the Root down to a specific node.
  • Height: The number of edges on the longest path from a specific node down to a Leaf.
  • Subtree: Any node inside the tree, together with all of its descendants, forms a Subtree.

Major Types of Trees

  • Generic Tree (N-ary Tree): A node can have any number of children.
  • Binary Tree: A strict rule where a node can have at most two children (Left and Right).
  • Binary Search Tree (BST): A Binary Tree with a sorting property: Everything in the left subtree is smaller than the parent, and everything in the right subtree is larger.
  • Self-Balancing Trees (AVL / Red-Black Trees): BSTs that automatically reorganize themselves during insertion to ensure the tree never becomes a skewed "straight line" (which would degrade search time to O(n)).
  • Trie (Prefix Tree): A specialized tree used heavily for auto-complete and string searching.
[Image of a Binary Search Tree data structure]

Time Complexity (Binary Search Tree)

The efficiency of a Tree depends entirely on its Height. If a tree is perfectly balanced, its height is $\log_2(n)$. If it is highly skewed (looks like a Linked List), its height is $n$.

  • Search: Average: $\mathcal{O}(\log n)$ | Worst (Skewed): $\mathcal{O}(n)$
  • Insert: Average: $\mathcal{O}(\log n)$ | Worst (Skewed): $\mathcal{O}(n)$
  • Delete: Average: $\mathcal{O}(\log n)$ | Worst (Skewed): $\mathcal{O}(n)$
